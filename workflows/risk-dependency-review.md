# Risk And Dependency Review

Workflow for reviewing risks, blockers, dependencies, owners, and next decisions across a program or product area.

## Objective

Make risk and dependency state visible without pulling raw ticket dumps into context.

## Default Mode

Read-only. Draft updates only.

## Output

- risk summary
- dependency map
- blockers
- owner/action table
- stale or unclear items
- decisions needed
- stakeholder questions
- memory proposal
