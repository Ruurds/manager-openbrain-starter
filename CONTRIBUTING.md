# Contributing to Manager OpenBrain Starter

Thanks for your interest. This is a practical scaffold, not a framework project — contributions should stay focused on making it more useful for managers in real work situations.

## What Good Contributions Look Like

- A workflow or prompt that solves a real management problem
- An example that makes an existing workflow clearer
- A schema improvement that makes outputs more structured or useful
- A fix for something broken or misleading in the docs
- A script improvement (better error handling, clearer output)

## What Does Not Belong Here

- Company-specific systems, credentials, or internal tool references
- Workflows that require proprietary or non-generic connectors
- AI hype or feature bloat — this repo values less over more
- Breaking changes to the core principles (read-only first, human approval, local-first)

## How to Contribute

1. **Fork the repo** — make your changes on your own copy
2. **Keep it focused** — one topic per pull request
3. **Test your workflow** — if you add a workflow or script, run it yourself first
4. **No secrets** — check that `.env`, tokens, API keys, and credentials are not included
5. **Open a pull request** — describe what problem it solves and how you verified it works

## What to Expect

- PRs are reviewed by the maintainer (currently one person)
- Feedback will be direct — scope, fit, and quality matter more than effort
- Not every PR will be merged — if something does not fit the neutral/generic scope it will be declined with a reason

## Principles This Repo Will Not Compromise On

- Read-only first
- Draft-only writes
- Human approval before external writes or memory stores
- No live credentials or client config in the repo
- Generic enough that any manager can use it without modification

## Questions

Open a GitHub Discussion or file an issue before building something large — it saves everyone time if the idea does not fit the scope.
