# Import Recipe Standard

This standard defines how future OpenBrain importers should be designed before any source-specific importer is added.

The goal is to avoid a memory swamp. Importers should bring in durable, approved context, not raw archives.

## Default Posture

All import recipes are local-first and dry-run first.

Default rules:

- no external writes
- no memory writes without explicit approval
- no external embedding APIs for confidential data
- no raw customer data, secrets, full ticket dumps, full email bodies, call recordings, or full transcripts by default
- source pointers and approved summaries are preferred over raw source copies
- imported items must pass validation before ingestion

## Required Importer Features

Every importer should support:

- `--dry-run`
- `--limit`
- `--after` or equivalent bounded source filter when the source has timestamps
- local sync log
- secret scan before embedding
- source metadata
- resumable execution
- batch progress reporting
- rate-limit/backoff handling where APIs are used
- import report output
- duplicate detection through `contentFingerprint`
- no memory writes unless explicitly run in apply/insert mode

## Import Flow

Use this sequence:

1. Discover source files or records.
2. Filter out unsupported folders, records, templates, generated files, and obvious noise.
3. Split source into durable memory candidates.
4. Classify sensitivity and allowed use.
5. Scan for secrets and prohibited data.
6. Summarize or redact where needed.
7. Compute source hash and content fingerprint.
8. Produce a dry-run report.
9. Ask for approval before memory insertion.
10. Insert approved items through the local memory ingestion path.
11. Write a sync log and final import report.

## Sync Log Contract

Each importer should keep a local ignored sync log. The sync log must not contain secrets or full source content.

Minimum fields:

```json
{
  "sourceId": "stable source record or file id",
  "sourcePointer": "human-usable pointer",
  "sourceTimestamp": "2026-05-01T09:00:00Z",
  "hashOfSource": "sha256-of-approved-source-slice",
  "contentFingerprint": "sha256-of-normalized-memory-text",
  "memoryId": "openbrain-memory-id-if-inserted",
  "status": "dry-run | inserted | skipped | rejected | needs-review",
  "reason": "short reason"
}
```

## Report Format

Each dry run should produce a short local report:

```markdown
# OpenBrain Import Dry Run

Source:
Date:
Mode: dry-run

## Summary

## Candidate Counts

## Rejected Items

## Duplicate Candidates

## Proposed Memory Items

## Approval Questions
```

## Rejection Rules

Hard reject:

- passwords, tokens, private keys, cookies, bearer values
- raw customer records
- call recordings or full call transcripts
- full confidential documents not explicitly approved
- raw Jira dumps or full issue comment histories
- full email bodies by default

Needs review:

- PII or regulated data
- source text with partial credentials or identifiers
- long summaries over the memory item threshold
- duplicate content fingerprint with a different source hash
- content that may supersede existing memory

## Candidate Importers

Useful future importers:

- sanitized Obsidian/Markdown notes
- approved meeting minutes
- ChatGPT export filtered to non-confidential durable decisions
- Jira/program summary reports already generated by OpenBrain
- local research memos

Avoid importing:

- raw mailboxes
- raw ticket/project exports
- unreviewed Slack exports
- broad cloud drive folders
- anything where retention and approval are unclear
